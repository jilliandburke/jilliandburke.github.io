What is it?

This piece of work was an assignment for my PHP class. We were given a sample employee database and asked to implement CRUD functionality and search on it, have authorization and use of a session variable, client-side validation in JavaScript and have an extra feature of our choice. I decided to add sort by employee ID as my extra feature as well as using Bootstrap and FontAwesome to add a bit of polish to the interface.

The Main View

To the right is the default view presented when the user logs in. This was made with using an HTML, Bootstrap and PHP. The buttons in the table will update or delete the corresponding record. Below the table are also next and previous buttons. There are currently twenty-five records per page. There is also a sort button so the user can sort either by ascending or descending employee ID. To the left is a navigation menu for the user to switch between views and tasks as well as a logout.

Add and Update Views

To the left are the add and update views. The update form contains the current information of the employee being updated. Though it is not displayed here, client-side validation is present on both of the forms and both use a date picker to ensure valid dates are entered.

Reflection

Overall, I am very proud of the assignment considering how much PHP, JavaScript and Bootstrap I knew at the time. If I were to redo this with my current knowledge there are definitely a lot of things I would do differently. For starters, I would use Laravel so I could have a more solid and secure login system, simpler pagination, more sorting capability and three-tier architecture. I would also use Bootstrap's column system to line things up better and have a nicer looking navigation menu that highlights what page the user is on. Logout would also be a seperate, smaller button located more out of the way of the main navigation menu.
